VEKTE USER DOCUMENTATION · 1.2

The complete manual for VEKTE — a deterministic 16-channel generative MIDI sequencer plugin (VST3 / AUv3 / standalone). Every lane, every knob, all 36 algorithms, MIDI routing, and channel-key sharing.

EXPLORE MANUAL
GETTING STARTED

Quick start

Get VEKTE running in your environment in under a minute.

  1. Insert VEKTE on a MIDI / instrument track (on iOS: load the AUv3 in your host, or run the VEKTE app).
  2. Start your DAW transport (or press Play in the iOS app).
  3. Turn on a lane (the colored square at a lane's left) — notes generate.
  4. Pick an algorithm (top-right dropdown of each lane) to change the pattern.
  5. Route VEKTE's MIDI to your synth(s) — see MIDI routing.

The host transport drives the plugin. The iOS standalone app opens stopped and has its own Play / Stop.

PITCH GENERATION

The NOTE lane

The NOTE lane generates pitched notes snapped to the chosen scale.

P1–P4 — the four algorithm knobs (the core)

The four highlighted ALGO knobs are the heart of VEKTE. They are the selected algorithm's own parameters, and together with the algorithm choice they are the deterministic source of truth for the pattern — same algorithm + same four values → the same 32 steps, every time.

They are renamed per algorithm to whatever that algorithm actually controls. For example: Euclidean → HITS · STEPS · ROT · FILL; Random Walk → STEP · RANGE · BIAS · DRIFT; Logistic Map → R · SEED · THRES · SPAN; Wave Interference → FRQA · FRQB · MIX · THRES; Flocking → AGNTS · COHS · SEP · ALIGN. All four exist on every lane, including the four MOD lanes.

NOTE LANE CONTROLS (DRAG VERTICALLY TO TUNE · ALGO = ALGORITHM PARAMETERS)
Knob Range Function
MIDI 1–16 Output MIDI channel for this slot.
STEPS 1–32 Playback window length over the 32-step pattern.
DENSE 0–100 Deterministic note-density sculptor (adds / removes on-steps).
STRUM −100…+100 Spread chord voices over time (needs a chord).
OCT+ / OCT− octaves Top / bottom of the note range.
BASE 0–24 Root note, snapped to the scale.
PROB 0–100 Probability each on-step actually fires.
ROT −16…+16 Rotate the pattern window (bipolar).
TIE% 0–100 Sustain length toward the next note (with TIE on).
SWING −100…+100 Delay or advance off-beat NOTE steps for groove (per channel).
HUMN 0–100 Humanize — deterministic timing / value jitter. Double-click to reset.
P1–P4 per algorithm The selected algorithm's four parameters — the deterministic core. Renamed per algorithm (HITS/STEPS/ROT/FILL, R/SEED/THRES/SPAN, FRQA/FRQB/MIX/THRES, AGNTS/COHS/SEP/ALIGN, …).

Determinism. The four algorithm knobs (P1–P4) + the algorithm choice are the source of truth. The same algorithm + knob settings always produce the same 32-step pattern. STEPS is a window over it, ROT navigates it, DENSE is a deterministic fill — all replicable across sessions.

Octave stretch. The algorithm's output is normalized to fill the chosen OCT−…OCT+ range and snapped to the scale, so the full range is always used. The range is inclusive (1.2): OCT− 3 / OCT+ 3 plays exactly octave 3; 3 / 4 plays two octaves.

Scale-locked editing. The piano-roll shows only in-scale rows, so notes you draw stay in key. Select Scale: NONE for free chromatic editing.

SNGL / DRM.

  • SNGL (Single): collapse notes to the base pitch; the chord still plays.
  • DRM (Drum): mono, base pitch only, no chord.

TIE. Notes sustain toward the next note by TIE% of the gap (0–100%). TIE is true legato: at 100% a note runs right up to the start of the next one — with no gap and no overlap — however that next note is swung, humanized or strummed. TIE% is dimmed when TIE is off.

Start anywhere. Begin playback, loop, or seek into the middle of a note and it sounds from that point, so a long tie or held note isn't silent until the next step. Change OCT, BASE or scale while a note is ringing and it re-pitches immediately rather than waiting for the next note.

Editing the piano-roll

Click / drag in the grid to toggle steps and set pitch (snapped to scale). The bottom strip shows the trigger pattern. The playhead sweeps in real time.

PROB and HUMN are shown ahead of the playhead

PROB doesn't roll the dice at the moment a step plays. The whole pattern's coin-flips are decided once per cycle, up front — so every probability-muted step is drawn on the grid (hollow) before the playhead reaches it, and the whole set re-rolls at the next cycle boundary.

HUMN (humanize) works the same way: its timing and value jitter is predetermined per cycle and drawn on the grid in advance. Both are deterministic, so what you see is exactly what plays — you can watch the next bar's ghosts and nudges before you hear them.

RND and the lane seed

Next to each lane's RND button is a small editable seed box. Pressing RND rolls a fresh seed and, from it, a new algorithm plus a new set of P1–P4 values for that lane. The seed is the short code for that roll.

  • Type a seed back in and the lane returns to that exact roll — same algorithm, same parameters.
  • The seed is a short alphanumeric code, and it is per lane — the NOTE lane and each of the four MOD lanes carry their own.
  • — means the lane has never been rolled.
  • A leading * means you have hand-edited the lane since it was rolled, so the seed no longer describes what's on screen.
  • Seeds save with your project, so a reopened session shows the same rolls.
Seed vs. key. A seed reproduces a randomize roll on your machine. To reproduce a pattern on someone else's, share the channel key — see Presets, sharing & keys.
CHORD VOICING

Strum

Spread a chord's voices across time instead of playing them all at once.

STRUM lives on the NOTE lane. Range −100…+100: positive strums low → high, negative high → low, and the spread scales with the note length. It is only active when a chord is selected — with no chord there is a single voice and nothing to spread.

STRUM is also available as a MOD target, so a MOD lane can sweep the strum over time for gestures that open and close, or flip direction mid-phrase.

MODULATION ENGINE

MOD lanes (1–4)

Each MOD lane runs its own algorithm to produce a smooth 0–1 value sequence, then writes it to a target.

Like the NOTE lane, every MOD lane has the same four algorithm parameters (P1–P4) — renamed to whatever its selected algorithm controls — plus MIN / MAX to set the output band the value sweeps within, and its own HUMN humanize.

CRV — smooth, seamlessly looping curves

CRV (curve) replaces the stepped values with smooth Catmull-Rom interpolation between mod steps — the way to get glides, filter sweeps, and slowly evolving modulation. MOD lanes free-run (no forced quantization), so the motion stays fluid.

The curve wraps around the loop point. Its end meets its beginning on the same trajectory — matching slope, not just matching value — so a CRV lane loops seamlessly whatever you set STEPS to. There is no jump, click, or sudden reversal when the pattern comes back around, whether the lane is 3 steps long or 32.

The curve drawn on the canvas is the curve that is emitted, including across that boundary.

When a MOD lane drives an internal control (octave, steps, scale, chord, etc.), the corresponding NOTE knob / dropdown shows the live modulated value.

PITCH — scale-snapped pitch bend 1.2

The Pitch target emits real MIDI pitch bend, up to ±1 octave. The lane's value picks a scale degree — the destination — and, with CRV on, the bend glides smoothly toward it. So the pitch always arrives in key, but you hear the slide getting there. Centre is no bend; the lane's MIN/MAX window how far it can reach.

Set your synth's bend range. VEKTE asks the synth for a 12-semitone wheel (via RPN 0) so a full sweep is an octave. Most synths honour this; if yours doesn't, set its pitch-wheel range to match the BENDRANGE value, or the sweep comes out the wrong size. Bend is per-MIDI-channel — it bends the whole chord together, like a real wheel — and re-centres automatically on stop.

RATCHET — rolls and stutters 1.2

Point a MOD lane at Ratchet and the held note retriggers for as long as it sounds — a roll, stutter or burst. The rate is an absolute musical value (1/4 · 1/4T · 1/8 · 1/8T · 1/16 · 1/16T · 1/32 · 1/64), so a 1/16 ratchet is a 1/16 whatever the lane's own division. Sweep the lane to move through the rates; the target label shows the rate it will actually play. Each hit is gated by LEN, and STRUM re-strums inside each hit — so a ratcheted chord rolls rather than machine-guns. The lane offers rates at least as fast as its own division, so the whole knob stays audible.

NOTE DIV — modulate the step rate 1.2

The Note Div target drives the NOTE lane's division, snapped to musical values, so a lane can speed the notes up and slow them down as it plays. Because the value snaps to exact divisions, the lane stays locked to the bar grid — the rate changes, the phase does not. A division change takes effect on the next step, so it never leaves the lane hanging.

HANDS-ON CONTROL · 1.2

MIDI Learn, Lock & keyboard control

Right-click any knob (or the algorithm dropdown) to open its context menu. From here you can bind the control to a MIDI CC, lock it against randomization, and more.

MIDI Learn — map a hardware knob to any control

Right-click a control → Learn CC, then move a knob or fader on your controller — VEKTE binds to the first CC it sees and that control now follows it live. Or type the number under Set CC# if you already know it. A small CC# badge above the knob shows the mapping.

  • Works on every knob, including the four algorithm parameters (P1–P4) on any lane.
  • One CC can drive many controls — learn the same CC on several knobs to gang them.
  • Bindings are per channel, so each of the 16 slots keeps its own controller map.
  • Mappings save with your project (not with the shared channel key — a pattern you send someone never carries your controller layout).
  • Clear CC removes a mapping.

The plugin must receive MIDI: enable your controller's input in the standalone's audio/MIDI settings, or route MIDI to the VEKTE track in your DAW.

Lock — protect a control from randomization

Right-click → Lock (or Alt-click the knob) to protect it. A locked control is skipped by RND and CHAOS but stays fully editable by hand; it greys out and shows a green padlock so you can see what's held.

  • Lock the algorithm: right-click the algorithm dropdown → Lock algorithm to keep a lane on its chosen generator through RND/CHAOS. Its P1–P4 stay put too.
  • Lock a single parameter: lock just one P-knob to freeze it while the rest of the algorithm re-rolls.

This per-control Lock is separate from the lane LOCK button, which freezes a whole lane's pattern.

Keyboard nudge

Click a knob or dropdown to select it — it gets a soft glow in the lane colour — then use the arrow keys (Up/Down or Right/Left) to step its value one increment at a time. Only one control is armed at once.

RHYTHMIC FEEL

Groove — swing & humanize

Add human micro-timing and stylistic pocket adjustments to your sequences.

  • SWING (per channel, −100…+100) — delay or advance the off-beat NOTE steps.
  • HUMN (humanize) on the NOTE lane and on each MOD lane — deterministic timing and value jitter. Double-click a HUMN knob to reset it.
  • SWNGLCK (swing lock) per MOD lane — see below.

SWNGLCK — lock a MOD lane to the swing

MOD lanes normally run straight, ignoring the channel's swing. Turn on SWNGLCK (next to CRV on any MOD lane) and that lane grooves in sync with the NOTE lane's SWING amount instead.

The important part: a locked lane still swings on its own step division (D:), not the NOTE lane's. So a 1/8 MOD lane and a 1/16 NOTE lane sit in the same pocket instead of fighting each other. With SWING at 0, the button does nothing.

Groove is predetermined per cycle

Swing, humanize and probability are all computed once per cycle for the whole pattern, then drawn on the canvas ahead of the playhead — nothing is decided at the last instant. The shifts and ghost notes you can already see for the rest of this cycle are exactly the ones you are about to hear, and the whole set refreshes at the next cycle boundary.

APPROACH & FILL — improvised passing notes 1.2

Two humanizers that bridge melodic jumps of two or more scale degrees with in-scale passing notes — deterministic, drawn on the piano roll before they sound, and always in key.

  • APPROACH — one to three short pickup notes just before the next note, only in the free space after the tie or gate ends — like a player leaning into the phrase.
  • FILL — the same walk stretched across the whole gap between two notes, each note sustaining its share: a melodic tie.
  • The amount (0–100) sets both the chance a gap is bridged and how many notes appear (1 → 2 from 50 → 3 from 85). The stronger of the two wins when both are set.
  • Drive them from a MOD lane (T: Approach / Fill) or host automation — there is no dedicated knob. HUMN above 50 arms a little APPROACH automatically.

MULTI-TIMBRAL SEQUENCING

Channels & slots

VEKTE is 16 channels wide, running 16 independent sequencers simultaneously.

The 16 buttons in the header are slots, and each slot is a complete, independent generator: its own NOTE lane, its own four MOD lanes, its own algorithms, scale, chord, groove, and output MIDI channel. Think of the 16 slots as 16 sequencers running at once inside one plugin, all sharing the host transport.

What a channel holds

Each of the 16 channels stores, independently of the others:

  • The NOTE lane — algorithm, P1–P4, STEPS, DENSE, PROB, ROT, octave range, BASE root, scale, chord, STRUM, TIE, SNGL / DRM, and any manual piano-roll edits.
  • Four MOD lanes — each with its own algorithm, P1–P4, STEPS, ROT, MIN / MAX, CRV, target (T:), and CC number.
  • Groove — swing, humanize, and per-MOD swing lock.
  • Playhead mode (Q) and step division (D:) per lane.
  • The output MIDI channel (1–16) the whole slot sends on.
  • Enable / lock / auto state per lane, and each lane's reproducible seed.

Everything above is what a channel key captures (see Presets, sharing & keys).

The live slot indicator

Every slot draws a small live readout even when it isn't selected:

  • A NOTE blink on each trigger, with a tie fade showing sustained notes.
  • Four mod-output bars — one per MOD lane — whose width tracks the live mod amount, so you can see all four modulators moving at a glance.
  • A slot with no active lanes is faded; a muted slot is struck through.

This lets you watch all 16 channels breathe from the header without opening each one.

Output channel & auto-grouping

Set a slot's output MIDI channel with the NOTE lane's MIDI knob (1–16). Slots that share the same output channel auto-group: they sit adjacently, get a faint surrounding box, and are labeled 1a / 1b / 2a / 2b ….

Grouping is how you layer — put two or more slots on one MIDI channel to stack parts on a single instrument (e.g. a bass line on 1a and its octave doubling on 1b, both hitting channel 1). Because each slot still has its own algorithm and groove, the layers can be rhythmically and harmonically independent while playing the same synth.

Channels are also modulation destinations. A MOD lane can target another lane's P1–P4, so with grouping + cross-lane targeting you can have one channel's modulator reshape a layered channel's pattern.
SEQ / KIT / ARP · 1.2

Slot modes & the slot bus

New in 1.2: every channel picks what it IS — a sequencer, a kit, or an arpeggiator — and channels can play each other.

The vertical rail on the NOTE lane's left switches the selected slot between three modes. Each slot shows its mode above the number in the header.

SEQ

The generative sequencer — everything described in the rest of this manual. This is the default.

KIT

A fixed one-octave kit: 12 notes mapped from the selected scale and root, mono, chords dormant. Entering KIT starts from no scale (chromatic — 12 semitones); pick a scale any time with S:. Point a KIT slot at a drum rack or sampler and let the algorithms play the pads.

ARP

Input-driven arpeggiation: the generated pattern plays only while notes come in, and the incoming note re-roots the whole pattern (the ROOT knob follows it live). Sound strictly follows the input — no held note, no output.

  • EXT / INT — where the input comes from: external MIDI, or the internal slot bus. VEKTE switches to INT automatically when another slot targets this one.
  • LATCH — the playhead keeps running silently while nothing is held, so the pattern re-enters mid-flow; without LATCH the playhead freezes on release and resumes where it stopped.
  • RTRG — every new press restarts the pattern from step 1 (a chord counts as one press).

The slot bus — channels playing channels

Turn a slot's DEST knob past 16 and it targets another slot (A1–A16) instead of a MIDI channel. A routed slot sends nothing to the MIDI output — every note it generates becomes the target ARP's input, root and velocity included. A SEQ conducting an ARP gives you melodies that play melodies; ARP-to-ARP chains work too, and cycles are rejected. DEST is also a mod target, so a MOD lane can re-route a slot per step.

OUTPUT CONFIGURATION

MIDI routing

VEKTE emits all 16 channels on one MIDI port, with per-channel routing.

macOS

VEKTE publishes a CoreMIDI virtual port named "VEKTE". In your DAW, set each destination track's MIDI From / input to VEKTE and choose the channel (1–16). In Ableton Live this is the reliable path — Live flattens track-to-track MIDI to channel 1, so the virtual port + the track's input channel filter preserves channels.

Windows

There is no built-in virtual MIDI. Install loopMIDI (free), create a port, then in VEKTE's Menu → MIDI Output choose that port. Route your DAW tracks from that port + channel filter.

Reaper / Bitwig / Cubase (chain-based hosts)

These pass MIDI channels through device chains — use the bundled Vekte2Live helper after VEKTE, or filter by channel on the receiving track.

Transport stop always sends clean note-offs.

Ableton Live — macOS vs Windows

Live is the one host that needs platform-specific handling, because Live's own track-to-track MIDI routing collapses everything to channel 1. VEKTE emits on 16 channels, so you must route around Live's internal flattening. The mechanism differs by OS:

Windows one-time setup: VEKTE's Windows installer bundles the system runtimes it needs; loopMIDI is a separate free download you install once.
TACTILE FEEDBACK

Visualization

The GUI draws real-time data to help you sculpt patterns visually.

  • Live piano-roll and bar / curve views with neon lane colors and a smooth 60 Hz playhead.
  • Gate length is drawn as a tail inside each note.
  • Probability-muted notes, swing and humanize are drawn ahead of the playhead for the whole current cycle — they're decided at the cycle boundary, not as each step fires.

Because every one of those is deterministic and computed a cycle at a time, the canvas is not an approximation: what you see is exactly what plays.

DATA RECONSTRUCTION

Presets, sharing & keys

Because VEKTE is fully deterministic, a pattern is a recipe, not a recording.

A given algorithm + its knob values always regenerate the exact same 32-step buffer. That is what makes sharing tiny: VEKTE only needs to store your inputs, then rebuild the notes on the other end.

Channel keys — share a whole channel as text

A channel key is a compact, copy-and-paste text string that captures one entire channel (all five lanes, groove, scale, chord, output channel, and your manual edits — everything listed in Channels & slots).

  • In the Menu → Copy Channel, the key appears in a select-all textbox with a Copy button.
  • It is short enough to drop into a chat message, a forum post, a caption, or a link. Send it to anyone.
  • They paste it into Menu → Paste Channel in their VEKTE and get your channel back identically — down to the note, on any machine or platform.

Why keys stay small. The key carries your parameter inputs and any manual piano-roll edits as a diff; the engine regenerates the algorithmic bulk locally. Two people with the same key hear the same thing because the generators are deterministic and versioned, not because the notes travelled in the key.

Compatibility & safety.

  • Older keys and saved files keep working — the format is backward compatible, so a key you shared last version still loads.
  • A key is validated on paste. If it references an algorithm a given build doesn't recognize, VEKTE rejects the key cleanly rather than loading a wrong or silently different pattern — so a shared key never mis-plays.
  • The five newest algorithms (Pack I, #31–#35) use integer-exact math, so keys that use them are identical across macOS, Windows, and iOS with no drift.

Three levels of recall

  1. Channel key — one channel, as text, for quick sharing.
  2. Song — all 16 channels together (key or file), for a whole setup.
  3. DAW project — full plugin state saves inside your session automatically; reopen the project and every channel, edit, and seed returns exactly.

Seed vs. key — two different things

Both reproduce something exactly, but they reproduce different things. Mixing them up is the one easy mistake here.

The key doesn't need the seed: it already stores the algorithm and parameter values that the roll produced, and regenerates the notes from those. So the channel key is the shareable artifact — the seed is a convenience for re-rolling your own lane.

Why a key grows — and when

A key's length is a direct readout of how much of the pattern is deterministic versus hand-made:

  • Purely generative channel → short key. If every note and value comes straight from the algorithm and its knobs, the key only has to carry those few inputs. The engine regenerates the 32 steps on the other end, so the string stays tiny.
  • Edited notes or values → larger key. The moment you draw or move notes on the piano-roll, or hand-edit mod values away from what the algorithm produced, those overrides can't be regenerated — they must be stored literally as a diff. The more steps you edit away from the deterministic output, the longer the key gets.

So a key that stays short means "all recipe"; a key that has grown means "recipe + your manual edits baked in." Both reload identically — the size just reflects how much you personalized by hand.

  • LOCK a lane you like, then RND / CHAOS the rest.
  • AUTO + a small ROT gives slowly evolving patterns.
  • Use CRV on a MOD → CC lane for smooth filter / parameter sweeps — it loops seamlessly at any STEPS setting.
  • Build variation layers by sending several slots to the same MIDI channel. They auto-group in the header (1a / 1b / 1c) and all play the same instrument, but each slot keeps its own algorithm, division, groove and probability. Instead of one thicker part you get several independent voices interlocking — a sparse high line over a dense low one, or the same motif at three different divisions. Mute individual slots (double-tap) to arrange on the fly.
  • Don't only route MOD lanes to CC. The most musical targets are often internal: point a MOD lane at Octave and a line leaps registers while staying in key; at Transpose for in-scale movement; at Scale or Chord to drift through a family; at Dense or Prob to breathe the rhythm in and out; at another lane's P1–P4 to reshape the generator itself.
  • Type a lane's seed back in to reproduce a roll you liked; seeds save with your project. To share the pattern with someone else, send the channel key instead.
